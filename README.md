# FFmpeg7-SDL2 多媒体实践项目

![FFmpeg](https://img.shields.io/badge/FFmpeg-7.0.2-green)
![SDL2](https://img.shields.io/badge/SDL2-2.30.6-blue)
![Platform](https://img.shields.io/badge/Platform-Windows-lightgrey)

</div>

基于 FFmpeg 7.0.2 和 SDL2 2.30.6 的多媒体播放器实践项目，专注于高性能音视频播放。项目经过重构，将核心功能提取到公共模块，实现了软硬件解码版本的音视频播放器，支持 4K 高帧率视频的流畅播放。

项目采用模块化设计，将公共代码提取到 common.h 和 common.cpp 中，提高代码复用率和可维护性。软硬件版本均具备基本播放功能、媒体信息显示，以及对不同分辨率与横竖方向视频的旋转伸缩适配。

博客：https://zhuanlan.zhihu.com/p/700478133

## 📦 依赖库
- **[FFmpeg 7.0.2](https://ffmpeg.org/)** - 音视频解码核心库
- **[SDL2 2.30.6](https://www.libsdl.org/)** - 跨平台多媒体渲染库

## 📁 项目文件说明

### 1. 📦 common.h - 公共头文件
**功能**: 定义共享的数据结构、常量和函数声明，为软硬件版本提供统一的接口。

**核心内容**:
- **线程安全队列**：`SafeQueue` 模板类，实现线程安全的数据传递
- **视频帧结构**：`VideoFrame` 结构体，用于存储视频帧数据和时间戳
- **公共常量**：队列大小、同步阈值等配置参数
- **全局变量声明**：FFmpeg 核心上下文、SDL 设备、线程和队列等
- **辅助函数声明**：文件选择、字符串转换等通用功能

**关键特性**:
- 提供统一的队列实现，确保线程安全
- 定义标准化的数据结构，简化代码复用
- 集中管理配置参数，便于统一调整

---

### 2. � common.cpp - 公共实现文件
**功能**: 实现 common.h 中声明的辅助函数，为播放器提供通用功能支持。

**核心内容**:
- **文件选择**：`SelectFileDialog` 函数，实现图形化文件选择界面
- **字符串转换**：`AnsiToUtf8` 函数，处理中文路径编码问题
- **全局变量定义**：初始化公共全局变量

**技术要点**:
- 使用 Windows API 实现文件选择对话框
- 处理 ANSI 到 UTF-8 的编码转换，确保中文路径正常工作
- 提供统一的辅助功能，减少代码重复

---

### 3. 🎯 Audio_Video_Player.cpp - 软件解码版本
**功能**: 基于软件解码的多线程音视频同步播放器，支持音视频文件播放，纯音频文件无窗口。

**核心特性**:
- ✅ **多线程架构**：解复用线程、音频解码线程、视频解码线程独立运行
- ✅ **线程安全队列**：使用 `SafeQueue` 实现无锁化的数据传递
- ✅ **智能流识别**：自动排除封面图片流，纯音频播放时不弹出视频窗口
- ✅ **精确同步**：以音频时钟为基准，动态调整视频渲染时机
- ✅ **实时进度显示**：在控制台显示当前播放进度
- ✅ **优雅退出**：支持窗口关闭退出，自动等待音频播放完毕
- ✅ **滤镜支持**：对不同分辨率和横竖方向的视频进行旋转伸缩适配

**数据流架构**:
文件 → [解复用线程] → 音频包队列 → [音频解码线程] → SDL_QueueAudio → 音频设备
                   → 视频包队列 → [视频解码线程] → 视频帧队列 → [主线程渲染] → 窗口

**技术要点**:
- 使用 FFmpeg 7.0 API 进行解码
- 音频重采样为 S16 立体声，视频统一转换为 YUV420P
- 音频时钟同步，确保音画同步
- 队列模式音频输出，实现简单可靠

---

### 4. ⚡ HW_Audio_Video_Player.cpp - 硬件加速版本
**功能**: 在软件版本基础上，增加 DXVA2 硬件解码支持，显著降低 CPU 占用，适合 4K/8K 高码率视频的流畅播放。

**核心特性**:
- ✅ **DXVA2 硬件加速**：自动检测并启用 DXVA2，将解码任务交给 GPU
- ✅ **多线程架构**：继承软件版本的高效多线程设计
- ✅ **智能内存管理**：采用有界队列和背压机制，防止内存爆炸
- ✅ **自动回退**：若硬件加速不可用，自动切换至软件解码
- ✅ **精确音视频同步**：沿用音频时钟为主同步策略
- ✅ **滤镜支持**：对不同分辨率和横竖方向的视频进行旋转伸缩适配

**技术要点**:
- **硬件加速初始化**：创建 DXVA2 硬件设备上下文并关联到解码器
- **硬件帧下载**：将 GPU 中的帧下载到系统内存，支持 YUV420P 和 NV12 格式
- **内存优化**：复用下载帧，减少内存分配
- **性能优化**：使用 SWS_FAST_BILINEAR 算法提高缩放速度

**性能对比**（以 4K 60fps 70Mbps 高压力视频测试为例）：
- 软件解码版本：CPU 占用 ≈ 30%，内存 ≈ 350MB
- 硬件加速版本：CPU 占用 ≈ 7%，内存 ≈ 400MB（增加约 50MB 用于下载帧，但换来了 4 倍的 CPU 性能提升）

**使用方法**:
1. **编译**：在 VS Code 中打开项目，选择要编译的文件（Audio_Video_Player.cpp 或 HW_Audio_Video_Player.cpp），然后运行构建任务
2. **运行**：编译完成后，运行生成的可执行文件
3. **选择文件**：程序会弹出文件选择对话框，选择要播放的媒体文件
4. **播放控制**：
   - 视频文件：会弹出播放窗口，显示视频画面
   - 音频文件：在控制台显示播放进度，无窗口弹出
5. **退出**：播放完毕自动退出，或点击窗口关闭按钮退出

## 🔧 常见问题

### Q1: 编译时找不到 FFmpeg/SDL2 头文件
**A**: 检查`.vscode/c_cpp_properties.json`中的 include 路径是否正确

### Q2: 运行时提示缺少DLL
**A**: 确保所有 DLL 文件在可执行文件同一目录，或已添加到系统 PATH

### Q3: PCM播放速度不对或有噪音
**A**: PCM_AudioPlayer.cpp只支持 44.1kHz / 16 位 / 立体声格式，请检查 PCM 文件参数

### Q4: 音频文件无法播放
**A**: AudioPlayer.cpp 支持主流格式，但某些特殊编码的文件可能需要额外的解码器

### Q5: 多线程播放器运行时 CPU 占用高？
**A**: 正常现象，多线程解码会充分利用 CPU。主循环中已添加 `SDL_Delay(10)` 避免空转，若仍过高可适当增大延时

### Q6: 硬件加速版本播放时内存比软件版本高？
**A**: 硬件加速版本会预先分配两个下载帧（约 20MB）用于从 GPU 获取数据，这是正常现象。相比软件版本，硬件加速版本 CPU 占用可降低 70% 以上，对于高分辨率视频是值得的。

### Q7: 硬件加速启用失败怎么办？
**A**: 程序会自动回退到软件解码，不影响播放。可检查显卡是否支持 DXVA2，或更新显卡驱动。
